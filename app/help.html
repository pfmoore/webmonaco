Monaco

This web page allows you to run the program Monaco that can be used to answer a wide range of probability problems, most commonly those arising in dice and card games.

Monaco is a program that you can run locally. However, that takes some setting up. This web page allows you to run the program remotely - even on a device such as a smartphone that you cannot run Monaco on directly. Most of the features of Monaco are available, the main exception being that run time is limited - but you can still do a lot in the time allowed.

An Example

We start with an example that describes how to use the web page. For this example we investigate the distribution of an ability score playing the game Dungeons & Dragons. We start with the usual way to produce such a score: roll four dice, pick the largest three and add them up. The most straightforward way to do this in Monaco is using the expression sum(tail3(sorted4d6)). For now we just assume that this expression works - where to find out how to write expressions, including that one, is described below.

To use that expression, we type it - or cut and paste it - into the box labelled Expression. We also select the buttons labelled -exact and -histogram at the top of the page, and then press the button Calculate below the three boxes.

This gives you 16 lines of output below that button - ignore the final output lines below the horizontal line, most users won't need those. The first line of that output is:

 3 -   1~ 0.000771605 = 1/1296

This line means that there is a probability of 1/1296, or about 0.000771605, of a score of 3. You might prefer to see that as a percentage. If so, also select the button labelled -percent before pressing Calculate. In this case the first line is:

 3 -   1~ 0.0771605% = 1/1296

Options

The three buttons labelled -exact, -histogram and -percent that have been used in the example above provide options to the program. There is also a box labelled Options; selecting the button -exact is exactly the same as typing -exact into that box. You can do one or the other, but trying to do both will result in the run failing.

There are many more options than those provided by buttons, which are just the most commonly used options. You can experiment with the other buttons for the example above - except that -probability is not appropriate, and at this point do not try using -new and always use -exact. The button Program Info actually selects three options. Some buttons are exclusive - for example selecting +statistics cancels -statistics, and vice versa.

Tutorial

There is a tutorial for the program, but to use it you need to understand the difference between how you would run a local copy of the program, and how you use this web page.

Monaco is a command line program. You run it by putting first the options, then the expression - and sometimes a third kind of parameter that the box Number is for - on a command line. Thus the first example above corresponds to the command line:

monaco -exact -histogram sum(tail3(sorted4d6))

where monaco is a placeholder for what you actually need to type, which depends on your system. The tutorial describes use of the program only in this way. To use examples in the tutorial you must put the options into the Options box - or use buttons where available - and put the expression into the Expression box. Until you reach the material on approximate runs near the end of the tutorial, leave the Number box empty. (When you reach that point, do not select -exact, put the number of evaluations in the Number box, and you can experiment with the option -new.)

The most important thing in the tutorial that is not either described above or can be found by experimentation with the buttons, is how to write useful expressions. However, the tutorial also describes some additional options, in particular how to use the option -probability.

The tutorial is one of the immediately available documents for the program, others include two further tutorials. For all of these see here. If you receive the program - contact the author as described there - there is also extensive full documentation - most of which most users will not need, the tutorials alone will probably suffice.
